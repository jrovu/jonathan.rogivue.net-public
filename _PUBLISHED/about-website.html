Processes

Initial Setup

  • Initialize content versioning in top level directory
    • git init
  • Commit changes & publish to remote repository

Create Content

  • On MacBook, Open Typora.app
  • Drag ~/Websites/(Sitename) folder into Typora
  • Pull down latest changes from Content Repository
  • Open Content To Publish
  • Prioritize the list, and start writing!

Publish the site

  • Terminal Commands
    • cd ~/Websites/(sitename)
    • (To Write) tools/markdown-to-site.rb
    • Converts directory of markdown to HTML
      • Combines navigation & styles
    • Opens homepage
    • (To Write) tools/publish-site.rb
    • Copies site to AWS S3 bucket using rsync
    • (To write) rsync ~/Websites/(sitename)/_PUB aws://(Bucket)
  • Convert Markdown to HTML: Pandoc
    • Inputs Markdown text, Outputs HTML text
      • `pandoc (input-filename).md -t html -s -o (output-filename).html
      • -t: To format (HTML)
      • (Include custom styles)
      • -s : Complete HTML page
    • Adds Navigation, Style sheets:
      • pandoc -B: Include content Before the main content (e.g navigation)
    • (Command to convert a folder of markdown files to an HTML site)
  • Clear cache
